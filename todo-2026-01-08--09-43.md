# GitLab + Vercel 部署计划

## 项目信息
- GitLab仓库：https://gitlab.com/wubinben/zero-to-ship
- 项目类型：Next.js 14.0.0
- 部署平台：Vercel
- 目标域名：免费域名 + 可选自定义域名

## 部署任务清单

### 第一阶段：GitLab仓库准备
1. [ ] 验证GitLab仓库访问（https://gitlab.com/wubinben/zero-to-ship）
2. [ ] 检查仓库权限设置（确保Vercel可以访问）
3. [ ] 确认主分支和项目结构
4. [ ] 验证.gitignore文件配置

### 第二阶段：Vercel账户和GitLab连接
5. [ ] 创建Vercel账户并登录
6. [ ] 在Vercel中连接GitLab账户
7. [ ] 授权Vercel访问GitLab仓库
8. [ ] 导入 `wubinben/zero-to-ship` 项目到Vercel
9. [ ] 验证GitLab仓库连接成功

### 第三阶段：项目配置和部署
10. [ ] 配置环境变量（DEEPSEEK_API_KEY等）
11. [ ] 设置构建配置（自动检测Next.js）
12. [ ] 执行首次构建和部署
13. [ ] 验证免费域名访问（zero-to-ship.vercel.app）
14. [ ] 测试GitLab推送自动部署功能

### 第四阶段：自定义域名配置（可选）
15. [ ] 在Vercel项目中添加自定义域名
16. [ ] 获取Vercel提供的DNS配置信息
17. [ ] 登录Namesilo域名管理后台
18. [ ] 配置DNS记录（A记录或CNAME记录）
19. [ ] 等待DNS传播和SSL证书签发
20. [ ] 验证自定义域名访问

### 第五阶段：环境变量和API测试
21. [ ] 在Vercel项目设置中配置环境变量
22. [ ] 配置DEEPSEEK_API_KEY（必需）
23. [ ] 配置DEEPSEEK_BASE_URL（可选）
24. [ ] 重新部署以应用环境变量
25. [ ] 测试deepseek-optimize API路由功能

### 第六阶段：验证和优化
26. [ ] 全面测试应用功能（页面、API）
27. [ ] 验证HTTPS访问和CDN性能
28. [ ] 测试国内访问速度
29. [ ] 创建部署文档和故障排除指南
30. [ ] 验证自动部署流程稳定性

## 环境变量配置
- DEEPSEEK_API_KEY: DeepSeek API密钥（必需）
- DEEPSEEK_BASE_URL: DeepSeek API基础URL（可选，默认：https://api.deepseek.com）

## 注意事项
1. GitLab在国内访问更稳定，适合国内用户
2. Vercel对GitLab的支持与GitHub基本一致
3. 自动部署、HTTPS、CDN等功能都保持不变
4. 部署流程基本不变，学习成本低
