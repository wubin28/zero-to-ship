根据你的需求，你可以在保持GitHub仓库状态不变的情况下，将代码同时推送到GitLab。以下是在WSL2 Ubuntu终端中的操作步骤：

## 方法一：添加独立的远程仓库（推荐）

这种方法可以让你分别管理GitHub和GitLab仓库[ref:1,6]：

```bash
# 1. 查看当前远程仓库
git remote -v

# 2. 添加GitLab为新的远程仓库（命名为gitlab）
git remote add gitlab git@gitlab.com:wubinben/zero-to-ship.git

# 3. 推送代码到GitLab
git push gitlab main  # 如果你的主分支是main
# 或
git push gitlab master  # 如果你的主分支是master

# 4. 如果想推送所有分支
git push gitlab --all

# 5. 推送所有标签（如果有的话）
git push gitlab --tags
```

以后如果想分别推送到不同仓库[ref:3]：
- 推送到GitHub：`git push origin <branch>`
- 推送到GitLab：`git push gitlab <branch>`
- 同时推送：`git push origin <branch> && git push gitlab <branch>`

## 方法二：配置一次性推送到多个仓库

如果你希望一次`git push`就推送到两个仓库[ref:4,7]：

```bash
# 1. 给origin添加第二个push地址
git remote set-url --add origin git@gitlab.com:wubinben/zero-to-ship.git

# 2. 查看配置
git remote -v

# 3. 以后执行git push时会自动推送到两个仓库
git push origin main
```

这样配置后，每次`git push origin`都会同时推送到GitHub和GitLab[ref:4]。

## 首次推送注意事项

如果是首次推送到GitLab，建议使用`-u`参数设置上游分支：
```bash
git push -u gitlab main
```

选择哪种方法取决于你的使用习惯：方法一更灵活可控，方法二更方便快捷[ref:6,7]。
