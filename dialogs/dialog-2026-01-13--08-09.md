## npx 是什么？

**npx** 是 **Node Package eXecute**（Node 包执行器）的缩写，它是一个用于执行 npm 包的命令行工具[ref:5,6]。

## 与 Node.js 和 npm 的关系

当你安装 Node.js 时，会自动附带安装 **npm**（Node Package Manager，包管理器）。而 **npx 从 npm 5.2.0 版本开始就自动捆绑在 npm 中一起安装**[ref:12,13,15]。

简单来说：
- 安装 **Node.js** → 自动获得 **npm** → 自动获得 **npx**（如果 npm ≥ 5.2.0）

## npm 和 npx 的区别

### npm（包管理器）
- **用途**：安装、更新、删除和管理项目依赖包[ref:2]
- **示例**：`npm install create-next-app` 会将包下载到本地

### npx（包执行器）
- **用途**：直接执行 npm 包，无需全局安装[ref:2,6]
- **优势**：可以运行一次性命令或工具，无需永久安装，减轻系统负担[ref:3,6,7]

## 为什么用 npx 创建 Next.js 项目？

当你运行 `npx create-next-app@latest` 时：

1. npx 会自动下载最新版本的 `create-next-app`
2. 执行它来创建项目
3. 执行完成后，这个工具不会占用你的全局安装空间[ref:9]

这样你每次都能使用最新版本的脚手架工具，而不需要手动更新全局安装的包[ref:3,9]。

总结：npx 是随 npm 一起安装的执行工具，让你能方便地运行 npm 包而无需全局安装[ref:12,15,17]。
